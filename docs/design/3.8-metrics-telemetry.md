# Core Runtime: Metrics and Telemetry

Reference core runtime: [3.0-core-runtime.md](3.0-core-runtime.md).
Reference invariants: [1.3-invariants.md](1.3-invariants.md).

## Scope

Defines the minimal metrics and telemetry required to validate runtime health, performance, and correctness without adding heavy dependencies.

## Principles

- Metrics collection must not allocate on the steady-state path.
- Use lock-free or per-thread counters with periodic aggregation.
- Avoid unbounded label cardinality.

## Required Metrics

Connection lifecycle:

- `connections_active`
- `connections_accepted_total`
- `connections_closed_total`
- `connections_rejected_total`

I/O flow:

- `bytes_read_total`
- `bytes_written_total`
- `read_backpressure_events_total`
- `write_backpressure_events_total`

Timeouts:

- `timeouts_idle_total`
- `timeouts_header_total`
- `timeouts_body_total`
- `timeouts_write_total`

Protocol errors:

- `protocol_errors_total{kind}`

Latency:

- `request_latency_ms_bucket` (fixed buckets, per protocol)

## Telemetry Events

Minimal event stream (sampled):

- Connection accept/close
- TLS handshake success/failure (if enabled)
- Protocol parse error (with error code)

Events must:

- Be rate-limited and sampled.
- Avoid logging secrets (TLS keys, headers, bodies).

## Aggregation Model

- Per-worker counters stored in fixed arrays.
- Periodic aggregation into a global snapshot buffer.
- Export to stdout or a small metrics endpoint (optional).

## Validation

- Tests that confirm metric counters are updated without heap allocation.
- Stress tests that ensure metrics collection does not degrade throughput.
