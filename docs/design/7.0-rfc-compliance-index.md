# RFC Compliance Index

Reference architecture: [2.0-architecture.md](2.0-architecture.md).
Reference invariants: [1.3-invariants.md](1.3-invariants.md).
Reference proxy design: [5.0-reverse-proxy.md](5.0-reverse-proxy.md).

## Purpose

- Track RFC-facing behavior by requirement, code path, and test coverage.
- Prioritize audits around wire correctness first, then broader state-machine coverage.
- Keep fixes aligned with the runtime invariant that the steady-state request path must not add per-request heap allocation.
- Treat this as an engineering index, not a certification claim.

Current status reflects the latest confirmed audit findings. Work-in-progress patches do not count as compliant until they are merged and retested.

## Status Legend

| Status | Meaning |
|--------|---------|
| `implemented` | Code path exists and the requirement has direct test coverage or strong localized evidence. |
| `partial` | Some implementation exists, but coverage is incomplete or the surrounding behavior has not been fully audited. |
| `known issue` | A confirmed behavior gap, regression risk, or non-compliant edge case exists. |
| `unknown` | Not yet reviewed in enough detail to assign confidence. |
| `intentional deviation` | Behavior differs from the RFC and the deviation is explicitly documented and accepted. |

## Deferred or Optional RFCs

These may become first-class audit targets later, but they are not in the first wave:

- RFC 6455 for WebSocket semantics if WebSocket support remains a product goal.
- RFC 8441 if HTTP/2 WebSocket bootstrapping becomes part of the supported feature set.
- RFC 9220 if WebSocket over HTTP/3 becomes part of the supported feature set. Flagged as an architectural constraint in `docs/design/8.0-h3-performance-plan.md` — the defer-until-FIN model doesn't fit WebSocket streams.
- RFC 9111 if caching behavior becomes a supported server or proxy feature rather than pass-through behavior only.
- RFC 6265 (HTTP State Management / Cookies) — swerver does not parse or set cookies today, so the RFC is not load-bearing. Track if cookie-aware routing or session affinity is added.
- RFC 9218 (HTTP/2 Extensible Priorities) — the current h2 stack ignores PRIORITY frames (`src/protocol/http2.zig`). Track if priority-based scheduling is added.
- RFC 9287 (QUIC Greasing) — greasing is implicit in the h3 control-stream parser's `else => {}` for unknown frame types. Not actively implemented as a sender but tolerant as a receiver.
- RFC 9221 (QUIC Datagrams) — not implemented. Track if unreliable datagram support (e.g. for DNS-over-QUIC or WebTransport) is added.
