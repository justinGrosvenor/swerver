# User Stories

Reference personas: [personas.md](../personas.md).

## platform-operator

- As a platform-operator, I need predictable latency under load so that SLOs remain stable during traffic spikes.
- As a platform-operator, I need clear failure modes and configurable timeouts so that incidents are diagnosable and bounded.
- As a platform-operator, I need TLS and QUIC to be optional so that deployment dependencies stay minimal.

## backend-developer

- As a backend-developer, I need a stable request/response API so that app logic is portable across protocols.
- As a backend-developer, I need zero-copy request parsing so that application throughput scales without tuning.
- As a backend-developer, I need protocol feature flags so that I can ship HTTP/1.1-only builds when needed.
- As a backend-developer, I need transparent WebSocket proxying so that real-time applications work through the reverse proxy.

## reverse-proxy-operator

- As a reverse-proxy-operator, I need to forward requests to upstream servers so that I can expose backend services through a single entry point.
- As a reverse-proxy-operator, I need connection pooling to upstreams so that backend connections are reused efficiently.
- As a reverse-proxy-operator, I need multiple load balancing strategies so that I can distribute traffic based on my requirements.
- As a reverse-proxy-operator, I need automatic health checks so that unhealthy backends are removed from rotation.
- As a reverse-proxy-operator, I need configurable retry logic so that transient failures are handled gracefully.
- As a reverse-proxy-operator, I need header manipulation so that I can add X-Forwarded-For and custom headers.
- As a reverse-proxy-operator, I need circuit breakers so that cascading failures are prevented.
- As a reverse-proxy-operator, I need upstream connection metrics so that I can monitor backend health and performance.
