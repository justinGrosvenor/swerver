# Core Runtime: Error Taxonomy and Shutdown Policy

Reference core runtime: [3.0-core-runtime.md](3.0-core-runtime.md).
Reference contracts: [1.4-contracts.md](1.4-contracts.md).

## Scope

Defines canonical error categories and the shutdown behavior for each error class.

## Error Taxonomy

### I/O Errors

- `io_transient`: temporary errors (EAGAIN, EINTR).
- `io_remote_closed`: peer closed connection.
- `io_local_resource`: local resource exhaustion (buffers, file descriptors).
- `io_fatal`: unrecoverable backend error.

### Protocol Errors

- `proto_parse`: invalid request framing.
- `proto_state`: invalid stream/connection state transitions.
- `proto_flow_control`: flow control violation.

### TLS Errors

- `tls_handshake`: handshake failure.
- `tls_alert`: TLS alert from peer.
- `tls_internal`: provider failure.

## Shutdown Policy

Mapping rules:

- `io_transient`: retry if safe; do not close unless repeated beyond threshold.
- `io_remote_closed`: transition to `draining` then `closed`.
- `io_local_resource`: apply backpressure; reject new work; close if thresholds exceeded.
- `io_fatal`: transition to `error`, close immediately.
- `proto_parse`: send error response where possible, then close.
- `proto_state`: send stream/connection error per protocol, then close.
- `proto_flow_control`: close connection with protocol error.
- `tls_handshake`: send alert if possible, then close.
- `tls_alert`: close immediately.
- `tls_internal`: close and emit telemetry.

## Error Reporting

- Error codes should be stable and minimal to avoid unbounded cardinality.
- Log only error class + code + connection id; avoid request data.

## Validation

- Unit tests that map error categories to shutdown transitions.
- Integration tests for protocol error paths (HTTP/1.1, HTTP/2, HTTP/3).
