# Core Runtime: Protocol Handoff and Zero-Copy Interfaces

Reference core runtime: [3.0-core-runtime.md](3.0-core-runtime.md).
Reference contracts: [1.4-contracts.md](1.4-contracts.md).

## Scope

Defines how the runtime hands data to protocol stacks and receives responses without copying.

## Request Handoff

Inputs to protocol stack:

- `read_buffer` slice(s) (multi-segment)
- `limits` (header/body caps)
- `connection metadata` (id, peer, protocol)

Outputs from protocol stack:

- `RequestView` with slices into the read buffer
- `ParseState` (complete/partial/error)
- `ErrorCode` for protocol-specific handling

Rules:

- Protocol parsers must accept multi-segment inputs without requiring compaction.
- Protocol parsers must return slices that remain valid until the runtime releases the buffer(s).
- The runtime owns the buffer lifetime; protocol code must not retain slices across polls unless pinned.
- For partial parses, the runtime retains the buffer(s) and resumes parsing on the next read.

## Buffer Pinning

- Pinning keeps a buffer handle alive across poll boundaries.
- Pinning must be explicit and bounded (per-connection limit).
- Pinned buffers must be released when the response is fully queued or on error.
- Default pinned-buffer limit is 2 per connection (one read, one write).

## Response Handoff

Inputs from router/application:

- `Response` metadata (status, headers)
- `body` as slice or stream descriptor

Runtime responsibilities:

- Encode headers into a write buffer (or vector of buffers).
- Use sendfile/zero-copy for large static bodies where possible.
- Apply write backpressure and queueing.

## TLS/ALPN Integration

- If TLS enabled, the TLS provider wraps the byte slices before protocol parsing.
- ALPN selection determines protocol stack selection.
- Cleartext path bypasses TLS and passes raw read buffer to parser.

## Error Handling

- Parse errors map to canonical protocol error codes and close behavior.
- Protocol errors must produce proper connection close semantics per protocol.

## Validation

- Tests that verify zero-copy slices are stable across parse boundaries.
- Tests that validate buffer release only after response is fully queued.
