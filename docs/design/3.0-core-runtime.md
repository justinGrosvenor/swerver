## Scope

This spec defines the core runtime primitives used by all protocol stacks and I/O backends: connection lifecycle, buffer ownership, backpressure, and timeout enforcement.

## Goals

- Ensure steady-state request handling does not allocate per-request heap memory.
- Enforce backpressure and timeouts consistently across protocols.
- Make connection state transitions explicit and auditable.
- Provide deterministic memory and resource bounds.

## Non-goals

- Implementing platform I/O backends.
- Implementing protocol parsing or TLS/QUIC logic.
- Defining application routing semantics.

## Components

### Connection Pool

- Preallocates a fixed number of `Connection` entries based on `max_connections`.
- Issues stable connection identifiers and indexes for event routing.
- Resets state on release; no per-connection heap allocation after startup.

### Connection State Machine

Allowed transitions:

```
accept -> handshake -> active -> draining -> closed
accept -> active -> draining -> closed
active -> error -> closed
handshake -> error -> closed
accept -> error -> closed
```

### Buffer Pool

- Owns fixed-size buffers (`buffer_size`) with a fixed count (`buffer_count`).
- Returns handles to buffers; runtime is responsible for releasing handles.
- Exhaustion is a backpressure signal (read/write pause).

### Backpressure

- Read and write buffering use high-water thresholds.
- When a connection exceeds thresholds, the runtime must stop scheduling reads/writes until drained.

### Timeouts

Timeout phases: idle, header, body, write.

- Timeouts are enforced by the runtime for every connection.
- Phase selection is driven by protocol stack state.

## Configuration

Core runtime configuration lives in `config.ServerConfig`:

- `max_connections`: upper bound for connection pool size.
- `buffer_pool`: buffer size and count.
- `backpressure`: read/write high-water thresholds.
- `timeouts`: idle/header/body/write timeouts.

## Validation

- Unit tests validate connection state transitions and timeout checks.
- Load tests validate backpressure and buffer pool exhaustion behavior.
