# Contracts

Reference architecture: [1.0-intro.md](1.0-intro.md).

## Platform I/O Backend

| Input | Description |
| --- | --- |
| Listener config | Bind address, port, backlog, and socket options |
| Read buffers | Fixed-size buffers owned by the runtime |
| Write buffers | Response buffers and file descriptors |
| Timeouts | Idle, header, body, and write timeouts |

| Output | Description |
| --- | --- |
| Read events | Completed reads with connection identifiers |
| Write completion | Completed writes with byte counts |
| Accept events | New connections with socket handles |
| Error events | I/O errors mapped to canonical categories |

## TLS Provider

| Input | Description |
| --- | --- |
| TLS config | Certificates, private keys, ALPN, and cipher policy |
| Connection handle | Socket handle or QUIC connection |
| Plaintext buffers | Input/output buffers for decrypted data |

| Output | Description |
| --- | --- |
| Decrypted bytes | Plaintext buffers ready for parsing |
| Encrypted bytes | TLS records ready for network writes |
| ALPN selection | Selected protocol for HTTP/1.1 vs HTTP/2 |
| Errors | TLS alert or handshake failure |

## HTTP/1.1 Parser

| Input | Description |
| --- | --- |
| Byte slices | Read buffer slices with request bytes |
| Limits | Header and body size limits |

| Output | Description |
| --- | --- |
| Request view | Method, path, headers, and body slices |
| Parse state | Complete, partial, or error |
| Error code | Canonical parse error identifier |

## HTTP/2 Stack

| Input | Description |
| --- | --- |
| Frame bytes | Streamed frame data from I/O |
| Settings | Initial peer settings |

| Output | Description |
| --- | --- |
| Stream events | Header and data events per stream |
| Flow control | Window updates and limits |
| Errors | Connection and stream error codes |

## QUIC/HTTP/3 Adapter

| Input | Description |
| --- | --- |
| UDP packets | Received datagrams |
| Transport params | QUIC transport parameters |

| Output | Description |
| --- | --- |
| Stream events | HTTP/3 request/response events |
| Packet writes | Datagrams for network output |
| Errors | Transport or application errors |

## Embedded API

| Input | Description |
| --- | --- |
| Server config | ServerConfig with protocol feature flags and resource limits |
| Router | Route registration for method/path handlers |
| Middleware | Pre/post hooks with optional route scoping |
| App state/services | Typed pointers provided by the embedder |

| Output | Description |
| --- | --- |
| Request view | Zero-copy `RequestView` exposed to handlers |
| Handler context | Access to params, response builder, and DI accessors |
| Response | `response.Response` with zero-copy body or arena-backed data |
