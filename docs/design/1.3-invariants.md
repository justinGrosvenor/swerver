# Invariants

Reference architecture: [1.0-intro.md](1.0-intro.md).

## Runtime

- The steady-state request path must not allocate per-request heap memory.
- Connection state transitions must be explicit and validated for every protocol.
- Backpressure must be enforced on reads and writes to prevent unbounded buffering.
- All timeouts (idle, header, body, write) must be configurable and enforced.
- The server must degrade gracefully to HTTP/1.1 when HTTP/2 or HTTP/3 is disabled.

## Protocols

- HTTP/1.1 parsing must reject invalid or ambiguous header framing.
- HTTP/2 stream state transitions must follow RFC-defined state machine rules.
- HTTP/3 connection-level flow control must prevent unbounded send queues.

## Security

- Request size limits must be configurable and enforced at parse time.
- TLS keys and secrets must not be logged.
- Protocol errors must close connections with the correct error codes.
